01문제 확인
인증 유저 기준 age = 40(36-40세)의 자연스러운 비율은 10.4%입니다. 반면 미인증 유저에서는 88.4%가 age = 40으로 기록되어 있습니다. 단순 연령 분포의 차이로 설명할 수 없는 수준으로, 시스템 기본값이 적용된 것으로 판단됩니다.
02데이터 명세서 확인
Membership_Description.xlsx  —  is_user_verified : 본인인증여부  (미인증시 성별/연령 부정확)
데이터 설계 단계에서 이미 미인증 유저의 age와 gender는 신뢰할 수 없다고 명시되어 있습니다. 즉, 이 현상은 데이터 품질 문제가 아닌, 시스템 설계상 의도된 동작입니다. 본인인증 절차를 거치지 않은 경우 서비스가 성별과 연령을 알 수 없어 기본값을 부여합니다.
03gender = N 기본값도 동반 발생
인증 유저의 gender = N 비율은 0.3%에 불과하지만, 미인증 유저에서는 67.3%가 gender = N으로 기록됩니다. age = 40과 마찬가지로 미인증 시 성별 기본값으로 N이 부여되는 구조입니다.
04원인 1 — iOS 앱스토어 결제 (billing_method = 140)
billing_method = 140은 Apple App Store 인앱결제 전용입니다 (payment_device가 100% ios). 애플 정책에 따라 앱스토어 구독 시 서비스 제공자에게 사용자의 성별·생년월일이 전달되지 않습니다. 따라서 해당 경로로 가입한 유저는 자동으로 is_user_verified = N, age = 40, gender = N으로 기록됩니다. iOS 미인증 유저 2,175명 중 98.1%가 age = 40입니다.
05원인 2 — 비-iOS 결제 후 본인인증 미실시
Android, PC 등 비-iOS 결제 수단으로 가입했으나 본인인증 절차를 완료하지 않은 경우에도 동일하게 age = 40 기본값이 적용됩니다. 비-iOS 미인증 유저 1,921명 중 77.4%가 age = 40입니다. iOS보다 비율이 낮은 이유는 일부 유저가 가입 시 직접 연령 정보를 입력했기 때문입니다.
06미인증 4,096명 케이스 분류
미인증 유저를 age·gender 신뢰 여부에 따라 네 케이스로 분류하면, 91.2%(A+B+C = 3,735건)가 age 또는 gender 중 하나 이상을 신뢰할 수 없습니다. 실제 정보가 있는 케이스 D는 361건(8.8%)에 불과합니다.

07기본값 제거 전/후 age 분포 비교
미인증 유저의 age를 결측치로 처리하면 age = 40의 과대 표현이 해소되고, 실제 이용자 연령대는 20-30대에 집중된 분포로 나타납니다.
08전체 구조 요약 — age = 40 구성 분해
전체 age = 40 레코드 5,104건 중 실제 36-40세는 1,399건(27.4%)에 불과합니다. 나머지 3,621건(70.9%)은 iOS 기본값 또는 비-iOS 미인증 기본값입니다.
09결론 및 모델링 처리 방향

원인 요약

미인증 유저(is_user_verified = N)에게 시스템이 자동으로 age = 40, gender = N을 기본값으로 부여합니다. 주요 경로는 두 가지입니다.

  1. iOS 앱스토어 결제 (billing_method = 140) — 애플 정책상 개인정보 미전달, 2,134건(52.1%)
  2. 비-iOS 결제 후 본인인증 미실시 — 인증 절차 생략, 1,487건(36.3%)

모델링 처리 권장사항

  1. is_user_verified = N인 경우 age, gender를 결측치(NaN)로 처리 — 기본값이 모델에 노이즈로 작용하는 것을 방지
  2. billing_method = 140 (iOS) 여부를 별도 피처로 추가 — iOS 유저의 행동 패턴을 독립적으로 학습
  3. gender = N도 결측치로 처리 — 미인증과 동일 논리 적용
  4. 인증 유저 13,487건의 age/gender만 신뢰하여 분석 및 학습
